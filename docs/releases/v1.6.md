# Release Plan: v1.6

Last Updated: 2026-04-01
Status: in_progress

## Goal
Add article status management with `draft` / `published` / `archived` across API and editor/list interactions.

## Scope In
- Data model adds `status` field for posts.
- Backend supports status read/write and list filtering.
- Public access only shows published posts by default.
- Editor can set status; list/detail can display status.

## Scope Out
- Image upload (v1.7)
- Multi-user role system
- Workflow-heavy moderation

## First Batch Tasks
- [x] Add `status` field to Post model with default `published`.
- [x] Add DB compatibility patch for existing SQLite files.
- [x] Add backend validation for allowed statuses.
- [x] Add list API status filter (`status=draft|published|archived`, login only for non-public states).
- [x] Restrict draft/archived post detail for anonymous users.
- [x] Add editor status selector.
- [x] Show status in list/detail cards.
- [x] Add richer status-specific empty-state copy.
- [x] Add dedicated tests for status visibility rules.

## Acceptance
- [x] Existing compose startup path still works.
- [x] `GET /api/posts` returns published posts for anonymous users.
- [x] Logged-in users can create/update post status.
- [x] Docs updated for v1.6 entry.

# Changelog

## Unreleased (v1.5.0)

### Added
- 全站统一的 info / loading / success / error 提示条
- 列表页、详情页、评论区与编辑页的轻量使用说明与可信提示
- 编辑页空预览状态提示，和其他空状态文案保持同一风格

### Changed
- 页面运行版本更新为 `v1.5.0`，用于当前版本验证
- 评论空状态、加载中提示与站内其他状态反馈风格统一

### Notes
- 当前已正式进入 `v1.5`，首批工作聚焦反馈样式、空状态文案和可信提示
- 本条目记录版本内迭代结果，待版本完成后再整理正式发布说明
### Internal fixes (smoke-test)
- 修复 Docker 运行时的 SQLite 问题：构建镜像时容器内 `backend/blog.db` 异常为目录，已删除并通过 `init_db.py` 初始化为 SQLite 文件，API `/api/posts` 返回 200（空集合），本地 Docker smoke-test 验证通过。

## v1.4.1 - 2026-03-31

### Added
- 编辑页双栏 / 单栏切换能力
- 固定在顶部的编辑工具栏与保存入口
- 编辑阶段的字数、字符统计与布局状态提示
- `Ctrl/Cmd + S` 快捷保存与离开编辑页前的未保存变更提醒
- 页面中的运行版本标识，方便部署后快速确认当前版本

### Changed
- 编辑区表单分组更清晰，标题、分类、标签与正文的层级更明确
- 移动端进入编辑页时默认使用更稳定的单栏布局

### Notes
- 当前已正式进入 `v1.4`，首批工作聚焦编辑效率与移动端编辑体验
- 本条目记录版本内迭代结果，待版本完成后再整理正式发布说明

## v1.3.0 - 2026-03-31

### Added
- 首页列表状态摘要、空状态说明与“已全部加载”提示
- 详情页标题区、文章元信息区与评论刷新入口
- 评论加载失败重试与成功态提示

### Changed
- 详情页长文阅读区域、代码块与表格在移动端的展示更稳定
- 评论输入区改为更完整的展开式交互，补齐提交中与取消状态
- API 针对文章不存在、评论不存在、未登录与登录失效场景统一返回文案

### Fixed
- 列表分页参数在异常输入下返回不稳定的问题
- 评论区加载失败后缺少明确反馈的问题
- 小屏设备下头部、详情页与评论区按钮容易拥挤的问题

## v1.2.0 - 2026-03-29

### Added
- 评论限流能力
- 列表排序能力

### Changed
- 排序控件从占位状态切换为可用状态
- 列表查询接口支持新的排序参数

### Fixed
- 评论高频提交缺少保护的问题
- 排序交互与后端返回不一致的问题

### Notes
- 当前部署方式已逐步切换到根目录统一 Docker 编排

## v1.1.0 - 2026-03-28

### Added
- bcrypt 密码校验
- JWT 密钥环境变量化
- 评论删除接口与前端删除能力

### Changed
- 登录表单不再预填默认密码
- 安全相关配置从硬编码调整为环境变量注入

### Fixed
- 明文密码暴露风险
- 评论缺少管理入口的问题

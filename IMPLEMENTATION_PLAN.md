# XiaozhiBlog 版本规划

**版本号规则**: `v主版本.功能版本.补丁版本`
- 第三位：发布后的补丁修复（如线上发现 bug → v1.1.1、v1.1.2）
- 第二位：完整功能交付（本文档规划范围，测试通过才打 tag）
- 第一位：大版本里程碑（暂无计划）

**提交与 tag 分离原则**：
- 开发过程中只做 `git commit`，不打 tag
- 功能全部完成 + 本地验收通过 → 打 `git tag v1.X.0`
- 上线后发现小问题修复 → 打 `git tag v1.X.1`，以此类推

## v1.3 - 体验优化版

**发布目标**: 改进分页体验，优化移动端

### 设计目标（本版本 UI 原则）

- 可读性优先：信息层级清楚，正文和交互按钮不抢焦点
- 触达效率优先：常用动作 1-2 次点击完成
- 移动端优先：手机端不拥挤、不误触、不横向滚动
- 反馈明确：加载、成功、失败都有可见状态

### 具体任务

- [ ] 列表页（首页）UI 优化
  - 分页机制改为"加载更多"（追加渲染，不覆盖原列表）
  - 列表卡片信息层级重排：标题 > 元信息 > 操作按钮
  - 筛选区折叠态和展开态视觉区分更明显
  - 空状态文案和入口按钮更清晰（例如返回默认筛选）

- [ ] 详情页 UI 优化
  - 标题、时间、阅读量与正文区的垂直节奏统一
  - 返回按钮和编辑按钮的层级区分（主次按钮样式）
  - 长文场景下的代码块、引用块间距优化

- [ ] 评论区 UI 优化
  - 评论输入区改为"展开式"（默认紧凑，点击展开）
  - 提交按钮状态：默认/提交中/提交失败/提交成功
  - 评论列表项增加更清晰的分隔和时间信息可读性

- [ ] 代码提交
  - commit message: `improve: 分页优化 + 移动端响应式`

### 定义完成

- ✓ 首页分页改为"加载更多"且最后一页自动隐藏按钮
- ✓ 详情页在长文情况下无明显布局跳动
- ✓ 评论提交全链路有状态反馈（提交中/成功/失败）
- ✓ 主要交互（搜索、查看、评论）可在 3 步内完成

### 页面级验收清单（测试打勾用）

- [ ] 首页：首屏加载完成后，用户可直接搜索并查看文章
- [ ] 首页：点击"加载更多"后内容是追加，不会跳回顶部

## 开发流程

1. 开发过程中只做 `git commit`，不打 tag
2. 某个第二位版本全部任务完成 + 本地验收通过 → `git tag v1.X.0` → 推送部署
3. 上线后发现问题 → 修复 → `git tag v1.X.1`，以此类推
4. 下一个功能版本开发时，重复第 1 步
